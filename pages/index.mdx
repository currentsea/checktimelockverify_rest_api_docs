# Introduction
Welcome to the documentation page for the [coinbin-checktimelockverify-rest-api](https://github.com/currentsea/coinbin_rest_api_service) project.

You can deploy this lightweight API using the [docker image](https://hub.docker.com/r/currentsea/coinbin-checktimelockverify-rest-api) if you would like something convenient.

**NOTE**: Please air on the side of caution and always run from source whenever possible, in order to do that you can reference the [public github](https://github.com/currentsea/coinbin_rest_api_service) for this lightweight API as an alternative to using a docker container like i show in this guide.

## What is a timelock?
According to the [official](https://en.bitcoin.it/wiki/Timelock) wiki for bitcoin, A **Timelock** is a type of smart contract primitive that restricts the spending of some bitcoins until a specified future time or block height.

You can read about [freezing funds](https://github.com/bitcoin/bips/blob/master/bip-0065.mediawikiuser-content-Freezing_Funds) for use in the future.
